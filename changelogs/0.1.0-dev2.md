# Changelog of 0.1.0-dev2

## Added

- Every world now runs on its own actor thread. Mutation inside a world -
  block placement/break, ticking, liquids, entities, movement, combat,
  effects, inventory changes, world local broadcasts - is sequential within
  that world and separate worlds run fully in parallel. Player and entity
  operations validate the target's current world membership and a binding
  epoch before applying, so a task queued against a world a player has since
  left produces no side effects instead of a stale mutation.
- Each world keeps its own player registry (session reference plus binding
  epoch) instead of asking the server for a list of everyone connected and
  filtering it down. Broadcasts, entity queries, and membership checks now
  resolve directly from the owning world.
- Each connected session has its own outbound queue and writer thread. A
  slow or stalled client's socket can no longer block a world's simulation
  thread or another player's delivery - packets are queued and sent
  independently per session. Batches of packets stay adjacent on the wire,
  a graceful disconnect drains anything already queued before closing, and
  a queue that fills up aborts that one session rather than dropping a
  packet silently or blocking the caller.
- Real Microsoft OIDC/RS256 login verification, replacing a check against
  the wrong key that was silently rejecting genuine Xbox-authenticated
  players. Includes live discovery/JWKS lookups and a from-scratch RSA
  signature verifier.
- A large set of deterministic concurrency tests covering per world
  isolation (ticking, liquids, entities, movement, combat, effects, block
  events), world-switch/epoch correctness and outbound delivery ordering
  and overflow handling.

## Changed

- The server's central `Hub` is no longer where gameplay execution happens.
  It's now a lifecycle and lookup surface - connected-session registry,
  world registry, cross-world lookup and global server configuration -
  not a shared actor every mutation had to pass through.
- Global admin/config state (difficulty, whitelist, operator status) moved
  from a job queue to direct, explicitly locked methods on `Hub`, now that
  there's no longer a single actor thread serializing everything by
  accident.

## Fixed

- Command packet builder returned a value instead of a pointer, so
  once packet delivery became asynchronous, the writer thread could end up
  reading a stack variable that no longer existed.
- Several cross world packet leaks, where an update in one world was
  broadcast to every connected player regardless of which world they were
  actually in (block updates, entity equipment, status effects).
- An outbound writer thread that never exited when a session with nothing
  queued was aborted, leaking one thread per disconnect.
- A disconnect that didn't actually stop a later packet from being queued
  behind it.
- A race between a session's own pre activation direct writes and the
  outbound queue taking over, which could let a packet be written twice or
  out of order.
